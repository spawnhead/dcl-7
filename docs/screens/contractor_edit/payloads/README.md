# Contractor Edit — Payloads

> Примеры запросов и ответов для редактирования контрагента.

## Files

| File | Description |
|------|-------------|
| `open-response.json` | Response for GET `/api/contractors/{ctrId}/edit/open` |
| `save-request.json` | Request for PUT `/api/contractors/{ctrId}/edit/save` |
| `save-response.json` | Response for PUT `/api/contractors/{ctrId}/edit/save` (success) |
| `network.har.BLOCKED.md` | Instructions for HAR capture in legacy (BLOCKED) |

## Usage

These payloads are used for:
1. Backend API implementation reference.
2. Frontend mock data during development.
3. Integration test fixtures.
4. QA verification.

## Validation Errors

For validation error responses, see CONTRACTS.md §2.

## HAR Capture

See `network.har.BLOCKED.md` for instructions on capturing legacy network traffic.
