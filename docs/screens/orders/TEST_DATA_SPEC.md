# Orders list — test data spec (fake dataset)

Источник: OrderFilterFakeProvider.buildRows(). Детерминированные данные для паритета и QA.

## Формат дат

- **Контракт (API):** `date_begin`, `date_end` — строками в формате **DD.MM.YYYY** (как в legacy).
- **Ответ API:** даты в полях `ord_date`, `ord_date_conf` и т.д. в формате ISO **YYYY-MM-DD**.
- **UI грид:** отображает даты в формате **DD.MM.YYYY** (как в фильтре).

## Покрытие по годам (2+ года)

| Диапазон ord_id | ord_date          | Кол-во строк | Назначение |
|-----------------|-------------------|--------------|------------|
| 1–45            | 2024-01-01 … 2024-01-28 (цикл) | 45 | Базовый набор 2024 |
| 46–55           | 2025-02-01 … 2025-02-10 (по дню) | 10 | Маяки для периода 01.02.2025–10.02.2025 |
| 56–70           | 2026-01-01 … 2026-01-09 (цикл) | 10 | Маяки 2026 |

**Всего:** 70 строк.

## Ожидаемые результаты фильтра по периоду

- **date_begin=01.02.2025, date_end=10.02.2025** → 10 строк (ord_id 46–55), все ord_date в [2025-02-01, 2025-02-10].
- **date_begin=01.02.2025, date_end=05.02.2025** → 5 строк (ord_id 46–50).
- **date_begin=01.01.2024, date_end=31.01.2024** → 28 строк (ord_id 1–28, даты 01.01–28.01.2024).
- **date_begin=01.01.2030, date_end=31.12.2030** → 0 строк (период не пересекается с датасетом).

Границы периода включительные: `ord_date >= date_begin` и `ord_date <= date_end`.
